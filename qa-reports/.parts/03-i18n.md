# 03 — i18n Scan

## Verdict: FAIL

Critical issues found:
- **19 nav.json keys missing from de.json and ka.json** (entire namespace empty)
- **Missing sync namespace** for 10+ SyncIndicator translation calls
- **Hardcoded user-visible strings** in multiple components not wrapped in `t()`
- **8 untranslated TODO placeholders** in routing views
- **toLocaleTimeString() without explicit locale parameter** in 2 components
- **toLocaleString() without explicit locale parameter** in 1 component
- **4 empty translation files** (auth.json, common.json, errors.json, help.json)

#### FINDING: [I18N8] toLocaleTimeString() without explicit locale parameter
- **severity**: MEDIUM
- **file**: `/Users/toko/Desktop/LiverRa/packages/app/src/emr/components/nav/SyncIndicator.tsx`
- **line**: 117, 131
- **description**: Two calls to `new Date().toLocaleTimeString()` without passing the current locale. These will use the browser's default locale, not the app's selected locale (en, de, ka). Both occurrences in SyncIndicator popover.
- **suggestedFix**: Pass locale from useTranslation(): `new Date(...).toLocaleTimeString(locale)`. Model: see CasesListView (line 230) for correct pattern.

#### FINDING: [I18N11] Form fallback strings in component props bypass translation
- **severity**: MEDIUM
- **file**: `/Users/toko/Desktop/LiverRa/packages/app/src/emr/components/admin/UserInviteModal.tsx` and similar admin components
- **line**: 67, 71, 76, 82, 87, 94, 100
- **description**: While the component correctly calls `t()` for translations, it also provides fallback hardcoded strings as the OR operand (e.g., `t('admin:invite.title') || 'Invite user'`). - **suggestedFix**: Remove fallback strings. If a key is missing, the app should return the key name (as the default does in TranslationContext) so missing translations are visible during QA. Alternatively, add a dev-only warning when a fallback is used.

## Summary

**Critical blockers (must fix before German/Georgian launch):**
1. nav.json completely untranslated in de/ka (19 keys, affects main navigation menu)
2. Missing sync.json namespace (10+ untranslated sync status strings)
3. LandingView entirely hardcoded (30+ strings, affects hero page visibility)
4. Empty auth.json, common.json, errors.json, help.json files
5. Untranslated RUO disclaimer and sign-in page in SigninView

**High-priority (affects user experience in de/ka):**
1. LayerToggle anatomy labels not translatable
2. FormErrorBoundary hardcoded messages
3. Admin form labels hardcoded or missing keys
4. TODO placeholder views

**Medium-priority (locale formatting issues):**
1. toLocaleTimeString() calls without explicit locale (2 instances)
2. toLocaleString() calls without explicit locale (1 instance)
3. Fallback strings masking missing translations

**Recommendation:**
- Allocate 2-3 business days for translation team to populate auth.json, common.json, errors.json, help.json, and complete nav.json for de/ka.
- Create dedicated sync.json or move sync keys to separate namespace.
- Extract all hardcoded strings from LandingView, SigninView, LayerToggle into translation files.
- Remove fallback hardcoded strings from component props.
- Add lint rule to flag missing locale parameters on toLocaleString/toLocaleTimeString.

# CT fixture placeholder — `ct-001.zip`

This directory is a **placeholder**. The real liver CT fixtures referenced by the
Playwright E2E tests (task T192) are **not committed** to the repo — they contain
PHI-adjacent imaging and exceed Git-friendly size limits.

## Expected fixtures

| Filename | Purpose | What it contains |
|---|---|---|
| `ct-001.zip` | Happy-path upload | A valid 4-phase liver CT: unenhanced, arterial, portal-venous, delayed. DICOM tree. ≥150 slices per phase. Fully anonymized (DICOM Basic Profile + option clean-descriptors). |
| `ct-missing-pv.zip` | Failure scenario | Same structure as `ct-001.zip` but with the portal-venous series deleted. Must trigger backend `422 missing_portal_venous_phase`. |

## How tests resolve the fixture path

Tests reference fixtures relative to the package root:

```
src/emr/views/__e2e__/liver-ai-pipeline/fixtures/ct-001.zip
```

At test time, a `globalSetup` (future task) copies the real archives from the
directory pointed to by the environment variable:

```
LIVERRA_E2E_FIXTURES_DIR=/path/to/private/liverra-e2e-fixtures
```

into this directory. If `LIVERRA_E2E_FIXTURES_DIR` is unset, tests SKIP with a
clear message instead of failing — CI environments without the private dataset
still get a green run for the non-upload scenarios.

## Acquiring the fixtures

The canonical fixture set lives in the LiverRa research S3 bucket
`s3://liverra-research-fixtures-eu-central-1/e2e/` (access-controlled).
Engineering: sync via `aws s3 sync s3://.../e2e/ $LIVERRA_E2E_FIXTURES_DIR`.

## Anonymization contract

Every fixture MUST pass CTP with the LiverRa profile before it lands in the
fixtures bucket. No PatientName, PatientID, InstitutionName, referring
physician, accession number, or acquisition-date-of-birth tags may be present.
DICOM UIDs are re-generated under the LiverRa UID root (`2.25.*`).

## Do NOT commit real DICOM data to this directory.

`.gitignore` already rejects `*.zip` and `*.dcm` under `fixtures/`.
